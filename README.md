# Telegram AI Бот для мыслей

Бот работает как простое место для мыслей, задач, идей, ссылок, фото и голосовых. Пользователь просто отправляет всё подряд в Telegram, а бот сам разбирает сообщение на короткие карточки.

## Что умеет

- Принимает текст, голосовые, аудио, фото, ссылки и пересланные сообщения.
- Голосовые расшифровывает локально через `faster-whisper`.
- Фото сохраняет в `data/photos`; AI может описать изображение, а Tesseract дополнительно распознаёт текст на фото.
- Автоматически пишет «Коротко», выделяет задачи и следующий шаг.
- Разбивает длинные хаотичные сообщения на отдельные карточки.
- Определяет тип записи: идея, задача, напоминание, контент, покупка, мысль, инсайт, ссылка, наблюдение.
- Ведёт раздел `Мысли` одним списком с пагинацией, поиск, экран `Продолжить мысль` с выбором карточки и архив.
- Каждую мысль можно открыть отдельно: продолжить, переименовать, сменить категорию, архивировать или удалить навсегда.
- Даёт альбом загруженных фото с возможностью удалить фото из альбома и локального хранилища.

## Быстрый старт локально

1. Установите Python 3.12. На Windows Python 3.14 не подходит для `faster-whisper`.
2. Создайте `.env` из `.env.example`.
3. Заполните `BOT_TOKEN` и `GROQ_API_KEY`.
4. Проверьте голос:

```env
VOICE_TRANSCRIBER=faster_whisper
WHISPER_MODEL=small
WHISPER_LANGUAGE=ru
```

5. Запустите:

```powershell
.\start-bot.bat
```

## Фото, Vision и OCR

Фото с подписью работает сразу: файл сохраняется локально, vision-модель описывает видимое на картинке, OCR подтягивает читаемый текст, а общий AI-разбор создаёт карточку. Если vision недоступен, бот продолжает работать по подписи и OCR без выдуманных деталей.

Для visual AI нужен `GROQ_API_KEY` и vision-модель:

```env
PHOTO_VISION_ENABLED=true
GROQ_VISION_MODEL=meta-llama/llama-4-scout-17b-16e-instruct
PHOTO_VISION_TIMEOUT_SECONDS=60
```

Для дополнительного OCR текста на фото нужен Tesseract:

```env
PHOTO_STORAGE_DIR=data/photos
PHOTO_OCR_ENABLED=true
TESSERACT_BINARY=tesseract
PHOTO_OCR_LANGUAGE=rus+eng
PHOTO_OCR_TIMEOUT_SECONDS=30
```

Если vision и Tesseract недоступны, бот не падает: он создаёт карточку по подписи. Если нет ни подписи, ни AI-описания, ни OCR-текста, бот сохраняет фото в альбом и создаёт короткую карточку `Фото без подписи`.

## Docker/Dokploy

Dockerfile использует Python 3.12 и ставит `ffmpeg`, `tesseract-ocr`, `tesseract-ocr-rus`. Для деплоя достаточно передать переменные из `.env.example`; модели Whisper хранятся в volume `/app/models`, данные и фото - в `/app/data`.

## Команды

- `/start` - первый экран.
- `/list` - мысли.
- `/next` - продолжить мысль.
- `/search запрос` - поиск.
- `/album` - альбом фото.
- `/archive` - архив.
- `/help` - как пользоваться.
- `/settings` - настройки.
- `/admin` - админка пользователей, доступна только администраторам.

## Проверка

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests
.\.venv\Scripts\python.exe -m compileall -q src tests
```

Важно: в текущем локальном `.venv` может быть Python 3.14. В таком окружении тесты проходят, но реальная загрузка модели Whisper на Windows блокируется защитой в коде. Для голосовых используйте Python 3.12 или Docker.
